Contenedores en docker

Los contenedores son instancias en ejecución de una imagen. Estos son entornos aislados que incluyen una copia del sistema operativo (SO), aunque comparten el kernel del SO del host, esto permite que las apps se ejecuten sin interferir con otras en el mismo host.

Características

  • ● Autónomo: Cada contenedor tiene todo lo que necesita para funcionar sin depender de dependencias preinstaladas en la máquina host.
  • ● Aislado: Dado que los contenedores se ejecutan de forma aislada, tienen una influencia mínima en el host y otros contenedores, lo que aumenta la seguridad de las aplicaciones.
  • ● Independiente: Cada contenedor se gestiona de forma independiente. La eliminación de un contenedor no afectará a ningún otro.
  • ● Portátil: El contenedor que se ejecuta en su máquina de desarrollo funcionará de la misma manera en un centro de datos o en cualquier lugar de la nube.

Creación de contenedores por comandos

En el apartado de Imágenes descargamos la imagen de MySQL de DockerHub, ahora crearemos el contenedor del servicio de MySQL con esa imagen usando este comando:

  • ● "--name mysql" : Especifica el nombre del contenedor que vamos a crear
  • ● "-d": Esto ejecuta el contenedor en modo “detached”, es decir, en segundo plano y sin bloquear la terminal actual
  • ● "-e MYSQL_ROOT_PASSWORD=1234 -e MYSQL_DATABASE=usuariosDB": Esto son argumentos para configurar variables de entorno, debes poner “-e” antes de cada variable.
  • ● "mysql": Esto especifica la imagen Docker que se utilizará para crear el contenedor, en este caso la img mysql que creamos anteriormente. En caso de no tenerla y que exista en DockerHub se descargará automáticamente

Una vez ejecutamos el comando, se crea el contenedor y nos devuelve su id.

Creación de contenedores por Docker Desktop

Iremos al apartado "Images" de DockerHub y buscaremos nuestra imagen, una vez la veamos le daremos al icono de la flecha.

Tras esto, nos saldrá un menú para crear nuestro contenedor. Debemos rellenar los datos como el nombre, puerto que queramos usar, si usamos volúmenes, variables de entorno... Una vez hayamos terminado de elegir la configuración le damos en "Run"
