Spring Security

Spring Security es un módulo del ecosistema de Spring para implementar seguridad en aplicaciones Java proporcionando mecanismos robustos.

1. Características

  • a) Autenticación:
    • Verificar la identidad de los usuarios, confirmando que son quienes dicen ser. Esto se logra mediante mecanismos como el uso de nombre de usuario y contraseña, tokens JWT, autenticación basada en redes sociales, entre otros.
  • b) Autorización:
    • Controlar el acceso a recursos y funcionalidades de la aplicación en función de los permisos y roles de cada usuario. Spring Security permite definir reglas granulares de autorización, permitiendo un control preciso sobre quién puede hacer qué dentro de la aplicación.
  • c) Protección contra ataques:
    • Spring Security incorpora medidas de seguridad para proteger las aplicaciones contra una amplia gama de amenazas, como ataques de inyección de código, ataques de fuerza bruta, falsificación de solicitudes entre sitios (XSS), entre otros.
  • d) Integración con Spring:
    • Al estar diseñado específicamente para el ecosistema Spring, Spring Security se integra a la perfección con otros frameworks y componentes de Spring, simplificando la implementación de seguridad en las aplicaciones.
  • e) Flexibilidad y personalización:
    • Spring Security es altamente flexible y personalizable, permitiendo adaptarlo a las necesidades específicas de cada proyecto. Ofrece una amplia gama de opciones de configuración y extensiones para cubrir diversos escenarios de seguridad.

2. Arquitectura Spring Security

  • - DelegatingFilterProxy:
    • Es un filtro de Jakarta Servlet que permite unir el ciclo de vida del contenedor de Servlet y el de Spring ApplicationContext dentro del FilterChain.
  • - FilterChainProxy:
    • Componente central de Spring Security que maneja la coordinación de las cadenas de filtros de seguridad.
  • - SecurityFilterChain:
    • Interfaz que representa una cadena de filtros de seguridad que se aplican a las solicitudes HTTP en una aplicación web.

3. Qué es JWT y cómo funciona

JSON Web Token (JWT), es un estándar abierto (RFC 7519) que define un formato compacto y autónomo para la transmisión segura de información. Consta de 3 partes:

  • - Header (Encabezado):
    • Tipo de token y algoritmo de firma.
  • - Payload (Carga útil):
    • Información deseada, como la identidad del usuario, a estos datos se le suelen llamar Claims
  • - Signature (Firma):
    • Verifica la integridad del token.
