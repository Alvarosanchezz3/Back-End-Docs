Spring

Spring Framework es un conjunto de herramientas para desarrollar aplicaciones Java empresariales. Ofrece características como Inversión de Control, Inyección de Dependencias, Aspect-Oriented Programming y módulos para diferentes capas de la aplicación. Facilita la gestión de transacciones y promueve la modularidad y la reutilización del código. Es muy utilizado en la industria debido a su robustez y facilidad de uso.

1. Crear proyecto Spring

Se puede crear yendo a https://start.spring.io/ o mediante el uso de plugins dentro de tu IDE. Una vez allí, deberás elegir:

  • 1. Herramienta de gestión de proyectos: Usaremos Maven
  • 2. Versión de Spring: Preferiblemente que no sea (SNAPSHOT o M1)
  • 3. Información del proyecto:
    • - Group: com.nombre_empresa
    • - Artifact: identificador-proyecto
    • - Name: Nombre Descriptivo del Proyecto
    • - Description: Descripción del Proyecto
    • - Package name: Nombre del paquete principal
    • - Packaging: JAR
    • - Versión de Java: Depende mucho del proyecto, pero preferiblemente la más nueva
    • - Dependencias: Debes escogerlas en función del proyecto que vas a realizar. Una vez creado el proyecto, puedes añadir o quitar dependencias en el archivo pom.xml

2. Arquitectura multicapa

La arquitectura multicapa o también conocida como arquitectura MVC (Modelo-Vista-Controlador) se organiza en capas para una mejor separación de responsabilidades y modularidad del código. Aquí está una descripción rápida de cada capa:

  • a) Capa de Entidad (Entity): Esta capa define las clases que representan objetos (usuarios, pedidos…). Estas clases suelen ser simples POJOs (Plain Old Java Objects) que contienen los campos y métodos necesarios para describir los datos.
  • b) Capa de Repositorio (Repository): La capa de repositorio es responsable de interactuar con la base de datos. Aquí se encuentran las operaciones de persistencia como guardar, recuperar, actualizar y eliminar datos.
  • c) Capa de Servicio (Service): La capa de servicio contiene la lógica de negocio de la aplicación. Aquí se implementan los casos de uso de la aplicación, coordinando las operaciones entre la capa de controladores y la capa de repositorio.
